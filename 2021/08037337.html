个人博客如何搭建（上）

hexo git
知识
发布日期:   2021-08-03
文章字数:   1.1k

前言

​

最近学习了很多攻略，也终于成功搭建了自己的博客。感觉也是很有用的，可以把平时生活中的感想和学习中的心得放在上面。我学习的方法主要是用了hexo,github以及git三种工具。具体的教程网上有很多，所以我就说说我的过程和遇到的一些问题。

​

一.环境搭建

​

1.nodejs下载安装

​

教程

​

这个教程说的比较好，跟着一路点确定就行了。之前看的有些没有说清楚，出了不少错误。主要问题是在配置环境变量时要分清楚用户变量和系统变量。最后，最好更改npm的镜像安装cnpm，会加快之后搭建博客的速度。

​

2.git下载安装

​

教程

​

这个教程感觉看不看都行，安装的时候一路也是点确定就行，最好验证一下即可。

​

3.配置淘宝云

​

使用win+r键，输入cmd进入命令符窗口，输入以下指令：

cnpm config set registry https://registry.npm.taobao.org

然后输入指令验证：

cnpm -v

有返回信息不报错就可以了

​

二.GitHub配置

​

①.注册和创建仓库

​

GitHub主页

​

右上角注册，注册时想好用户名，因为如果不绑定域名的话你的网页地址就会和你的用户名有关。

注册登录后在右上角点击头像，选择setting，然后选择 Repositories，将main改为master。

​

之后，在右上角选择 new repository ：

​

之后，方框内填入 “自己的用户名”.github.io 即可，可以勾选 README 选项，然后点create创建。

​

②.添加ssh密钥

​

1.首先检测本机是否存在ssh密钥：

$cd ~/.ssh#

如果存在可跳过第二步。

2.输入指令，用自己绑定的邮箱替换：

$ssh-keygen -t rsa -C "your_email@example.com"

一直回车，直到出现以下提示：

Your identification has been saved in /c/Users/you/.ssh/id_rsa.
Your public key has been saved in /c/Users/you/.ssh/id_rsa.pub.
The key fingerprint is:
01:0f:f4:3b:ca:85:d6:17:a1:7d:f0:68:9d:f0:a2:db your_email@example.com

3.输入指令复制ssh密钥：

clip < ~/.ssh/id_rsa.pub

进入GitHub的setting中，点击ssh keys，选择add ssh key，粘贴进去add即可。

4.输入指令验证：

$ssh -T git@github.com

只要看到以下信息就算可以了：

Hi username! You’ve successfully authenticated, but GitHub does not provide shell access.

​

三.博客搭建

​

链接

​

直接在链接中找到 install.sh 然后下载，然后在想要存放博客的盘里新建文件夹将其放入。然后在该文件夹右键空白运行 git bash,分别运行以下命令：

source install.sh test_node
source install.sh test_git

出现版本号分别代表nodejs,git安装成功没有问题，然后输入指令：

source install.sh hexo_win

等待片刻，出现 Please run hexo s to check it out! 代表成功。

然后输入

cd hexoblog
hexo s

可以看到出现地址 http://localhost:4000/ ，选择后右键 open 即可在浏览器看到自己初始化的博客。

然后关闭git ，在存放install的文件夹再右键打开，输入：

source install.sh deploy

将新产生的 up.sh 文件放入hexoblog 文件夹中，然后在该文件夹中右键打开git，输入

source up.sh

然后在该文件夹中找到配置文件_config.yml，找到deploy，一般在最后，进行修改：

deploy:
  type: git
  repo: 这里填你的仓库地址,建议使用SSH地址,它是以git开头的
  branch: master

​

这样自己的博客就搭建好了，在浏览器输入仓库地址如 nanshu001.github.io就能访问到。当然这只是一个架子，想要让博客变得漂亮好看需要配置主题和美化。这些我们留到下篇再说。

​

注：遇到的问题

1.安装失败

最好用管理员模式运行 命令提示符 和 git ,可以找到其运行图标的所在位置，右键 属性 ，然后选择用管理员模式运行，以后也都会默认用管理员模式运行。有很多安装失败的原因都是因为这个。

此外，使用npm开头的命令失败时可以在前面加个c，即cnpm，就可以了。
